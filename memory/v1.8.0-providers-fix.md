When updating to v1.8.0, the `providers/` directory (containing `_http.mjs`, `_types.js`, `ashby.mjs`, `greenhouse.mjs`, `lever.mjs`) was not shipped by `update-system.mjs`. The scan.mjs static import crashed immediately.

**Fix:** Created `providers/` with all 5 files from the GitHub repo.

Also fixed `greenhouse.mjs` to not crash when an entry has `api:` pointing to a non-greenhouse ATS. The original `resolveApiUrl()` checked `entry.api` first and called `assertGreenhouseUrl()` on it unconditionally — Lever API URLs in the `api` field would throw. Fixed by only checking `entry.api` if it contains a known greenhouse hostname.

**Why:** The provider-based scan in v1.8.0 replaced the old hardcoded ATS detection. With 105 tracked companies but only 1 (Shield AI via Lever) matching a provider, the zero-token scan covers very few entries. Most companies need agent-based WebSearch scanning.

**How to apply:** If scan.mjs crashes on startup, check `providers/` exists. If greenhouse detect() throws on a company with a non-greenhouse api URL, this fix is already in place.
